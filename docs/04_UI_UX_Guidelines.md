# TutorTrack UI / UX Guidelines

**Project Name:** TutorTrack

**Version:** 1.0

**Status:** Planning

**Author:** Michael Davila

**Last Updated:** July 19, 2026

---

# 1. Design Philosophy

TutorTrack should feel calm, modern, and easy to navigate.

Tutors should spend their time thinking about missionaries, not figuring out how to use the software.

Every screen should communicate the most important information quickly without unnecessary visual clutter.

---

# 2. Core Principles

### Clarity

Important information should be easy to find.

Avoid crowded layouts.

---

### Simplicity

Keep interactions straightforward.

Avoid unnecessary clicks.

---

### Consistency

Buttons, forms, spacing, and typography should remain consistent throughout the application.

---

### Speed

Frequently used actions should require as few clicks as possible.

---

# 3. Layout

Every page should follow the same basic structure.

----------------------------------------------------

Top Navigation Bar

----------------------------------------------------

Page Title

Short page description (optional)

----------------------------------------------------

Main Content

----------------------------------------------------

---

# 4. Navigation

Primary navigation should include:

- Dashboard
- Districts

Navigation should always remain visible.

Users should always know where they are within the application.

---

# 5. Forms

Forms should be clean and easy to complete.

Guidelines:

- Labels above fields
- Large text areas for notes
- Logical grouping of related information
- Auto-save after changes
- Display a small "Saved" confirmation after successful updates

---

# 6. Cards

Cards should be used to display:

- Districts
- Companionships
- Missionaries

Cards should:

- Have subtle borders
- Include adequate spacing
- Highlight when hovered
- Be clickable

---

# 7. Teacher View

Teacher Views should:

- Be read-only
- Be printer-friendly
- Avoid unnecessary navigation
- Display only information relevant to teachers

The design should be simple enough to review quickly before class.

---

# 8. Colors

The application should use a clean, neutral color palette.

Accent colors should be used sparingly to highlight important actions and status information.

Examples:

- Follow-up due today
- Recently updated
- Success (Saved)

---

# 9. Responsive Design

TutorTrack should work well on:

- Desktop
- Tablet

Mobile optimization is not a priority for Version 1 but layouts should remain usable on smaller screens.

---

# 10. Accessibility

The application should:

- Use readable font sizes
- Provide sufficient color contrast
- Support keyboard navigation where practical
- Use descriptive labels for inputs and buttons

Accessibility should be considered throughout development.
